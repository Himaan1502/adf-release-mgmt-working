# ADF Working Repository - CI/CD Framework

> **Azure Data Factory** | ARM Template Export | GitHub Actions | Dev → UAT → Prod

This is the **working repository** for an ADF project. It holds the environment
parameter files and calling workflows that reference the shared reusable
deployment templates. The actual ARM template logic lives in the
[Release Management Templates repo](<YOUR_ORG>/<YOUR_RELEASE_MGMT_REPO>).

---

## 📁 Repository Structure

```
.
├── .env/
│   ├── UAT.txt                          # UAT environment parameter overrides
│   └── Prod.txt                         # PROD environment parameter overrides
│
├── .github/
│   └── workflows/
│       ├── validation.yml               # PR validation → runs against DEV ADF
│       ├── uat_deploy.yml               # Manual promote: DEV → UAT
│       └── prod_deploy.yml              # Manual promote: DEV → PROD
│
├── ArmTemplate/
│   ├── README.md                        # ARM template guidance
│   ├── ARMTemplateForFactory.json       # Auto-generated by ADF Studio (root)
│   ├── ARMTemplateParametersForFactory.json
│   └── linkedTemplates/
│       ├── ArmTemplate_0.json           # Linked template shards (auto-generated)
│       ├── ArmTemplate_1.json
│       └── ArmTemplateParameters_master.json  # Master parameter file (DEV defaults)
│
├── scripts/
│   └── README.md                        # Utility scripts (pre/post deployment hooks)
│
├── docs/
│   └── README.md                        # Architecture decisions, runbooks
│
├── .gitignore
└── README.md                            # ← You are here
```

## CI/CD Pipeline Overview

```
┌─────────────────────────────────────────────────────────────┐
│                        DEVELOPER                            │
│  Edit pipelines in ADF Studio (Dev) → Publish → adf_publish │
└─────────────────────────┬───────────────────────────────────┘
                          │  ARM templates committed
                          ▼
┌─────────────────────────────────────────────────────────────┐
│                   PULL REQUEST to main/uat/dev              │
│   workflow: validation.yml                                  │
│   → Validates ARM template against DEV ADF instance         │
└─────────────────────────┬───────────────────────────────────┘
                          │  PR merged
                          ▼
┌─────────────────────────────────────────────────────────────┐
│               MANUAL TRIGGER: uat_deploy.yml                │
│   → Uploads linked templates to Non-Prod blob storage       │
│   → Stops UAT triggers                                      │
│   → Deploys ARM with .env/UAT.txt overrides                 │
│   → Restarts UAT triggers                                   │
└─────────────────────────┬───────────────────────────────────┘
                          │  UAT sign-off
                          ▼
┌─────────────────────────────────────────────────────────────┐
│               MANUAL TRIGGER: prod_deploy.yml               │
│   → Uploads linked templates to Prod blob storage           │
│   → Stops PROD triggers                                     │
│   → Deploys ARM with .env/Prod.txt overrides                │
│   → Restarts PROD triggers                                  │
└─────────────────────────────────────────────────────────────┘
```

## GitHub Actions Secrets Required

Configure these at the **repository** or **organization** level:

| Secret Name | Description |
|---|---|
| `AZURE_CLIENT_ID` | Service Principal or OIDC App Client ID |
| `AZURE_CLIENT_SECRET` | SP secret (skip if using OIDC federated identity) |
| `AZURE_TENANT_ID` | Azure AD Tenant ID |
| `AZURE_SUBSCRIPTION_ID` | Non-Prod Subscription ID |
| `AZURE_SUBSCRIPTION_ID_PROD` | Prod Subscription ID |

> **Recommended:** Use OIDC Federated Identity instead of client secrets.
> See [Azure OIDC with GitHub Actions](https://docs.microsoft.com/en-us/azure/developer/github/connect-from-azure).

## Environment Parameter Files (`.env/`)

Each `.env/<ENV>.txt` file contains key=value pairs that **override** the
default parameters in `ArmTemplate/linkedTemplates/ArmTemplateParameters_master.json`
at deployment time. No ARM template modification is needed per environment.

| File | Target ADF | Subscription |
|---|---|---|
| `UAT.txt` | `DF-<ProjectName>-Tst-DA` | Non-Prod |
| `Prod.txt` | `DF-<ProjectName>-Prod-DA` | Prod |

## Linked Services Parameterized

The following linked services are parameterized across environments:

- **Azure Databricks** (main, APS, CMM clusters)
- **Azure SQL Database**
- **Azure Key Vault**
- **Blob Storage** (ARM template artifact staging)
- **Shared Integration Runtime** (if cross-factory IR is used)
- **Logic App URLs** (Prod only, if applicable)
- **OFS API connections** (Prod only, if applicable)

## Deployment Checklist

### Before UAT Deploy
- [ ] ARM templates published from Dev ADF Studio
- [ ] `ArmTemplate/linkedTemplates/` committed to `main`
- [ ] `.env/UAT.txt` values verified (cluster IDs, server names, KV URLs)
- [ ] UAT GitHub Actions secrets configured
- [ ] Non-Prod runner group (`Non-Prod`) is online

### Before Prod Deploy
- [ ] UAT deployment validated and signed off
- [ ] `.env/Prod.txt` values verified
- [ ] Prod SAS token is valid and not expired
- [ ] Prod GitHub Actions secrets configured
- [ ] Prod runner group (`Prod`) is online
- [ ] GitHub Environment protection rule (approval gate) configured on `Prod`
